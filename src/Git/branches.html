Git Branches

Navegação e Gerenciamento de Branches

Mudar para uma branchcls específica

git checkout [nome-da-branch]

Muda para uma branch específica no repositório. Se a branch não existir, o Git irá avisar que ela não foi encontrada.

Criar e mudar para uma nova branch

git checkout -b [nome-da-branch]

Cria uma nova branch e muda para ela ao mesmo tempo. Esse comando é útil para criar novas funcionalidades ou corrigir bugs em paralelo ao desenvolvimento principal.

Comando alternativo para mudar de branch

git switch [nome-da-branch]

Alternativa ao comando git checkout para mudar de branch. O comando switch é considerado mais intuitivo e foi introduzido em versões mais recentes do Git.

Voltar para um estado anterior do repositório

git checkout [commit]

Reverte o repositório para um estado anterior especificado por um commit. O repositório volta para o estado exato de como estava nesse commit.

Reverter um arquivo para o estado do último commit

git checkout -- [arquivo]

Reverte um arquivo para o estado do último commit, descartando as alterações feitas no arquivo desde o último commit.

Listar branches

git branch

Lista todas as branches locais no repositório. A branch atual será marcada com um asterisco (*).

Excluir uma branch

git branch -d [nome-da-branch]

Exclui a branch especificada. O Git irá avisar se há commits que ainda não foram mesclados, prevenindo a exclusão acidental de trabalho não mesclado.

Forçar exclusão de uma branch

git branch -D [nome-da-branch]

Força a exclusão da branch especificada, mesmo que haja commits não mesclados. Use com cuidado, pois isso pode levar à perda de trabalho.

Renomear uma branch

git branch -m [nome-atual] [novo-nome]

Renomeia a branch atual ou uma branch especificada para um novo nome.

Mesclar uma branch

git merge [nome-da-branch]

Mescla a branch especificada com a branch atual. Esse comando é usado para combinar o trabalho realizado em diferentes branches.
