Git Stash

Comando git stash

O comando git stash é usado para armazenar temporariamente alterações no seu diretório de trabalho (working directory) que ainda não foram commitadas. Ele permite que você "guarde" essas mudanças em uma "pilha" (stash) para que você possa retornar a um estado limpo do repositório e, mais tarde, recuperar essas alterações quando necessário.

Quando Usar git stash?

É útil quando você está trabalhando em uma funcionalidade ou correção de bug, mas precisa interromper seu trabalho temporariamente (por exemplo, para mudar de branch ou atualizar seu repositório local), mas não quer fazer um commit ainda. O git stash permite que você volte para um estado limpo do repositório e depois recupere as alterações quando for conveniente.

Como Funciona o git stash?

  • Armazenar alterações: Para guardar as alterações no stash e limpar seu diretório de trabalho, use: 
    git stash

    Isso armazena as modificações no stash, retornando seu repositório ao estado do último commit.

  • Exibir alterações armazenadas: Você pode visualizar os itens armazenados no stash com: 
    git stash list

    Isso mostra a lista de stashes criados, com o índice de cada um (como stash@{0}, stash@{1}, etc.).

  • Recuperar alterações armazenadas: Para aplicar as alterações do stash de volta ao seu diretório de trabalho (sem remover o stash da pilha), use: 
    git stash apply

    Caso você queira aplicar um stash específico (caso haja mais de um), você pode usar:

    git stash apply stash@{n}Onde n é o índice do stash.
  • Aplicar e remover o stash: Se você quiser aplicar as alterações do stash e, ao mesmo tempo, removê-lo da pilha (como se estivesse "usando" o stash), use: 
    git stash pop

  • Descartar alterações armazenadas: Caso você queira excluir um stash específico, sem aplicá-lo, utilize:

    git stash drop stash@{n}
  • Limpar todos os stashes: Para excluir todos os stashes de uma vez, use: 
    git stash clear

Exemplo de Fluxo de Trabalho com git stash:

  1. Você está em uma branch de desenvolvimento e fez algumas alterações que ainda não quer commitar.
  2. Você precisa mudar para outra branch rapidamente, então usa: 
    git stash
  3. Após terminar o trabalho na outra branch, você pode retornar à branch original e aplicar as mudanças armazenadas: 
    git stash pop

Opções Úteis com git stash:

  • git stash save "mensagem": Salva as alterações com uma mensagem descritiva para fácil identificação mais tarde.
  • git stash -u ou git stash --include-untracked: Além dos arquivos modificados, inclui também os arquivos não rastreados (arquivos novos que ainda não foram adicionados ao índice).
  • git stash -a ou git stash --all: Inclui arquivos ignorados (arquivos listados no .gitignore) além dos modificados e não rastreados.

O git stash é uma ferramenta poderosa para gerenciar mudanças temporárias, permitindo que você seja flexível e trabalhe de maneira mais eficiente sem perder seu progresso em andamento.
