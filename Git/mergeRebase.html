Git Merge e Rebase

Explicações dos Comandos merge e rebase

Comando merge

O comando git merge é utilizado para integrar mudanças de diferentes branches em uma única branch. Este comando toma o conteúdo da branch de origem e mescla-o com a branch alvo, criando um commit de merge se houver mudanças em ambas as branches.

git checkout main
git merge feature

Explicação:

  • git checkout main: Troca para a branch main.
  • git merge feature: Mescla as mudanças da branch feature na branch main.

Comando rebase

O comando git rebase é utilizado para mover ou re-aplicar commits de uma branch em outra. Este comando cria uma sequência de commits aplicando-os na base da branch de destino. O rebase pode tornar o histórico do Git mais limpo e linear.

git checkout feature
git rebase main

Explicação:

  • git checkout feature: Troca para a branch feature.
  • git rebase main: Re-aplica os commits da branch feature na base da branch main.

Diferenças Entre merge e rebase

  • merge: Mantém a história do commit intacta e cria um commit de merge.
  • rebase: Reescreve a história dos commits, aplicando-os em uma nova base, resultando em um histórico linear.

Quando Usar Cada Comando

  • Use merge quando a preservação da história do commit e dos commits de merge for importante.
  • Use rebase quando você deseja um histórico de commits mais limpo e linear, sem os commits de merge.
