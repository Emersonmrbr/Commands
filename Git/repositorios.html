Repositórios

Trabalhando com Repositórios

Inicializar um Repositório Git

git init

Inicializa um novo repositório Git no diretório atual, criando um repositório vazio para gerenciar o versionamento dos arquivos no local.

Verificar o Estado do Repositório

git status

Exibe o estado atual do repositório, indicando quais arquivos foram modificados, quais estão preparados para commit e quais ainda não estão rastreados pelo Git.

Trabalhando com Repositórios Remotos

Vincular um repositório local a um repositório remoto 

git remote add origin

Associa um repositório remoto a um repositório local, permitindo o envio e recebimento de alterações.

Renomear uma branch

git branch -m

Renomeia a branch atual. Esse comando pode ser útil para organizar ou corrigir erros nos nomes das branches.

Enviar alterações para o repositório remoto

git push origin

Envia as alterações da branch local para o repositório remoto, permitindo que outros colaboradores acessem as mudanças.

Baixar alterações do repositório remoto

git pull origin

Baixa as alterações da branch especificada do repositório remoto para o repositório local.

Obter informações sobre o repositório remoto

git remote -v

Exibe as informações do repositório remoto associado ao repositório local, como a URL do repositório.

Remover um repositório remoto

git remote remove

Remove um repositório remoto associado ao repositório local, caso não seja mais necessário.

Sincronizar branch local com o repositório remoto

git fetch origin

Baixa as alterações de um repositório remoto sem integrar essas mudanças ao repositório local. Isso ajuda a verificar o estado remoto sem afetar o repositório local.

Clonar um repositório remoto

git clone

Cria uma cópia local de um repositório remoto, incluindo seu histórico de commits e branches.

Gerenciamento de Conflitos

Detectar conflitos de merge

git merge

Quando você tenta fazer o merge de uma branch em outra, pode haver conflitos, que são alterações incompatíveis. O Git irá alertá-lo e marcar os arquivos com conflitos. Você precisará editar os arquivos para resolver os conflitos antes de realizar o commit.

Resolver conflitos de merge

Após detectar conflitos, abra os arquivos afetados e procure pelas marcações do Git que indicam as áreas conflitantes. Resolva o conflito manualmente e remova as marcações. Após isso, adicione os arquivos ao índice e faça o commit.

git add 
git commit

Cancelar um merge em andamento

git merge --abort

Se você decidir que o merge não é necessário ou que não deseja continuar devido a conflitos, pode cancelar o processo de merge e voltar ao estado anterior.
